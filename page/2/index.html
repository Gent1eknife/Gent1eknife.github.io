__toString() //把类当作字符串使用时触发 echo，正则匹配，字符串拼接等都会触发

__invoke() //当尝试将对象调用为函数时触发

__clone 调用clone方法时候调用

省流版：

当你打开系统代理时，代理软件就会修改系统设置中的代理选项。但是，系统设置中的代理往往只可设置为HTTP代理，而且这个HTTP代理只对当前系统上web浏览器有效。**这也就是为什么明明设置了系统代理，访问GitHub也很快，但在命令行里克隆一个GitHub的仓库却很慢的原因。**在命令行下，执行git命令也就是直接执行git的二进制文件，跟你设置的系统代理是没有关系的。

而 Tun 模式：

工作原理
Tun 模式通过在操作系统中创建一个虚拟网络接口（即虚拟网卡），实现对所有出站网络流量的捕获和重定向。这是通过修改系统的路由表完成的，将所有或特定目标的IP流量都重定向到虚拟网卡。接着，代理程序从这张虚拟网卡读取流量，处理后再发送到原本的目标地址或其他指定的代理服务器。

此模式的关键在于，它在操作系统层面拦截网络流量，不依赖于应用程序的代理支持，因此可以处理从 TCP 到UDP 的所有类型的流量。

最终解决方案：

开虚拟网卡模式，关闭系统代理，开局域网连接 wsl 开镜像模式，正常网页和外网和 wsl 代理都搞好了。

常见汇编记录

发表于 2025-03-29 Waline： 本文字数： 950 阅读时长 ≈ 3 分钟

MOV

  • 语法: MOV dest, src
  • 描述: 将数据从源操作数复制到目标操作数
  • 示例: mov eax, 0x10 → 将0x10写入eax寄存器
  • PWN场景: 初始化寄存器或内存数据

LEA

  • 语法: LEA dest, [address]
  • 描述: 计算有效地址并存储到目标寄存器（不访问内存）
  • 示例: lea rdi, [rbp-0x10] → 计算rbp-0x10地址存入rdi
  • PWN场景: 定位栈/堆内存偏移（如计算缓冲区地址）

PUSH/POP

  • 语法: PUSH src / POP dest
  • 描述: 将数据压入栈顶或弹出栈顶数据
  • 示例: push rax → 将rax压栈；pop rbx → 弹出到rbx
  • PWN场景: 保存寄存器状态、构造ROP链

CALL

  • 语法: CALL address
  • 描述: 调用函数（将返回地址压栈并跳转）
  • 示例: call 0x401000 → 调用函数0x401000
  • PWN场景: 劫持返回地址（如栈溢出覆盖返回地址）

RET

  • 语法: RET
  • 描述: 从栈顶弹出返回地址并跳转
  • 示例: ret → 返回到调用者
  • PWN场景: ROP链的结尾指令，控制程序流

JMP

  • 语法: JMP address
  • 描述: 无条件跳转到指定地址
  • 示例: jmp 0xdeadbeef
  • PWN场景: 跳转到shellcode或ROP gadget

CMP

  • 语法: CMP op1, op2
  • 描述: 比较两个操作数（设置标志位）
  • 示例: cmp eax, 0x20 → 比较eax和0x20
  • PWN场景: 条件判断（如检查输入是否合法）

TEST

  • 语法: TEST op1, op2
  • 描述: 对两个操作数做按位与（设置标志位）
  • 示例: test al, al → 检查al是否为0
  • PWN场景: 条件分支（如判断是否为空）

JE/JZ

  • 语法: JE/JZ address
  • 描述: 相等/为零时跳转（依赖ZF标志位）
  • 示例: je 0x404000 → 若相等则跳转
  • PWN场景: 绕过条件检查或触发漏洞

JNE/JNZ

  • 语法: JNE/JNZ address
  • 描述: 不相等/非零时跳转（依赖ZF标志位）
  • 示例: jne 0x404000
  • PWN场景: 类似JE，用于条件分支逻辑

ADD/SUB

  • 语法: ADD dest, src / SUB dest, src
  • 描述: 加减运算（修改目标寄存器）
  • 示例: add rsp, 0x20 → 栈指针增加0x20
  • PWN场景: 调整栈指针或寄存器值

XOR

  • 语法: XOR dest, src
  • 描述: 异或运算（常用于清零寄存器）
  • 示例: xor eax, eax → eax=0
  • PWN场景: 清零寄存器或加密解密操作

AND/OR

  • 语法: AND/OR dest, src
  • 描述: 按位与/或操作
  • 示例: and al, 0x0f → 保留低4位
  • PWN场景: 数据掩码或权限检查

INT 0x80 / SYSCALL

  • 语法: int 0x80 (x86) / syscall (x64)
  • 描述: 触发系统调用
  • 示例: syscall → 执行系统调用
  • PWN场景: Shellcode中调用execve等系统调用

LEAVE

  • 语法: LEAVE
  • 描述: 恢复栈帧（等价于mov esp, ebp; pop ebp）
  • 示例: leave → 退出函数时恢复栈
  • PWN场景: 栈溢出漏洞利用的关键点

NOP

  • 语法: NOP
  • 描述: 空操作（占位符）
  • 示例: nop → 无操作
  • PWN场景: 构造NOP sled（滑板指令）

MOVSx (MOVSB/MOVSW/MOVSD)

  • 语法: MOVSB
  • 描述: 按字节复制内存（从ESI到EDI）
  • 示例: rep movsb → 重复复制字符串
  • PWN场景: 内存操作漏洞（如缓冲区溢出）

CALL [reg]

  • 语法: CALL [reg]
  • 描述: 调用寄存器指向的地址
  • 示例: call rax → 跳转到rax中的地址
  • PWN场景: 劫持函数指针或虚表调用

LOOP

  • 语法: LOOP address
  • 描述: ECX递减，若不为零则跳转
  • 示例: loop 0x401000
  • PWN场景: 循环逻辑（如解密代码）

RETF

  • 语法: RETF
  • 描述: 远返回（从栈中弹出CS和EIP）
  • 示例: retf → 切换代码段
  • PWN场景: 某些特殊架构的漏洞利用

关键知识点总结：

  1. 栈操作（PUSH/POP/LEAVE）是栈溢出的核心。
  2. 控制流指令（CALL/RET/JMP）用于劫持程序执行。
  3. 系统调用（INT 0x80/SYSCALL）是Shellcode的必备。
  4. 比较与跳转（CMP/TEST/JE）用于绕过保护逻辑。
  5. 数据操作（MOV/XOR）用于构造Payload。
